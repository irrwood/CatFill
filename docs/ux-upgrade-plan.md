# CatFill UI/UX 升级计划

> 目标版本：v0.3.x
> 更新日期：2026-07-13
> 状态：P0 主流程和 P1 信息架构已完成；高级恢复能力继续迭代

## 1. 升级目标

CatFill 已具备学习、填写、AI 填写、资料导入、字段整理和多档案能力，但当前界面仍按技术功能组织。升级后的产品应按用户任务组织，让用户无需理解“本地匹配、字段信号、AI 整理”等内部概念也能完成工作。

核心目标：

1. 新用户在第一次打开后 30 秒内知道如何建立资料。
2. 老用户打开 popup 后只需一次点击即可开始填写。
3. 填写结果必须可信，明确区分成功、未匹配和失败。
4. AI 是可选增强，不阻挡本地基础能力。
5. 资料始终可查看、修改、删除，并清楚说明数据去向。

## 2. 目标用户旅程

### 旅程 A：首次使用

1. 打开 CatFill。
2. 选择建立资料的方式：学习当前网页、导入文件、手动添加。
3. 查看提取结果并确认保存。
4. 返回网页开始填写。

成功标准：用户不需要先配置 API Key，也不需要理解“字段整理”。

### 旅程 B：日常填写

1. 选择当前档案。
2. 根据风险选择“AI 安全填写”或“快速本地填写”。
3. 小猫书写动画反馈正在处理。
4. 显示实际填写数量和未匹配数量。
5. 结果明确标记本次使用了 AI 还是仅使用本地匹配。

成功标准：常规表单一次点击完成；零填充时绝不显示成功。

### 旅程 C：学习网页内容

1. 用户先在网页填写正确内容。
2. 点击“学习并保存本页内容”。
3. CatFill 显示新增、更新、忽略的结果。
4. 有歧义或敏感内容时要求确认。

成功标准：用户清楚哪些内容被保存，重复学习不会制造重复字段。

### 旅程 D：维护资料

1. 从 popup 打开“管理我的资料”。
2. 按类别查看已保存资料。
3. 直接修改值；需要时展开字段标题、别名等高级信息。
4. 删除或整理重复资料。

成功标准：常用修改无需进入编辑模式；技术元数据默认隐藏。

### 旅程 E：启用 AI

1. 用户在遇到复杂表单或文件提取时看到 AI 能解决什么。
2. 选择服务商并填写 API Key。
3. 测试连接并明确显示连接状态。
4. 保存后返回原任务继续操作。

成功标准：设置过程可中断、可恢复，用户知道 Key 保存位置和发送范围。

## 3. 信息架构调整

### Popup：只处理当前网页

- 当前档案
- 推荐操作：AI 安全填写
- 快速操作：快速本地填写（不调用 AI，仅适合字段明确的表单）
- 资料操作：学习并保存本页内容
- 填写结果与后续动作
- 底部入口：管理我的资料
- 设置齿轮

保留三个入口，但清楚说明风险和数据流。AI 填写是推荐主操作；本地填写作为明确标注的快速模式，不允许用户误以为它包含语义审题。

### Side Panel：管理长期资料

三个一级 Tab：

1. 我的资料
2. 导入资料
3. 设置

“整理重复资料”只在“我的资料”中有可整理内容时出现，不再作为全局顶栏主按钮。

## 4. 文案体系

| 当前文案 | 建议文案 | 原因 |
|---|---|---|
| 当前联系人 | 当前档案 | 档案不一定属于联系人，也可能是公司或家人 |
| 一键填写（本地匹配） | 快速本地填写 | 明确不调用 AI，只适合字段清晰的表单 |
| AI 智能填写 | AI 安全填写 | 强调会先理解问题和格式 |
| 学习本页已填内容 | 学习并保存本页内容 | 明确学习会产生保存行为 |
| 字段整理 | 我的资料 | 使用用户语言，不使用内部概念 |
| 添加资料 | 导入资料 | 与“我的资料”形成清楚分工 |
| AI 整理 | 整理重复资料 | 说明实际效果 |
| 确认保存 | 保存这些修改 | 说明保存对象 |
| 忽略 | 暂不处理 | 降低不可逆感 |
| 已填充 7/10 个字段 | 已填写 7 项，还有 3 项未匹配 | 直接说明下一步问题 |

文案原则：

- 按钮使用“动词 + 对象”。
- 状态说明发生了什么，不描述内部技术阶段。
- 错误后必须提供下一步。
- 中文和英文表达自然对应，不做逐字翻译。

## 5. 分阶段实施

### P0：重构 Popup 主流程

- [x] 保留三个入口，并明确区分 AI、本地与学习三种行为。
- [x] AI 填写作为推荐主操作，本地填写明确标注不调用 AI。
- [x] 档案为空时，把主区域替换为首次使用引导。
- [x] “打开字段整理侧栏”改为“管理我的资料”。
- [x] 填写期间显示小猫奋笔疾书动画。
- [x] 完成动画只在实际填写数量大于 0 时出现。
- [x] 支持夜间模式和 `prefers-reduced-motion`。

验收：用户无需理解本地匹配与 AI 的区别；一次点击即可完成常规填写。

### P0：可信的结果反馈

- [ ] 返回并展示 `filled`、`unmatched`、`skipped`、`failed` 四类结果。
- [x] 零填充使用中性提示，不显示完成对勾。
- [x] 显示“用 AI 继续”等上下文操作。
- [x] 学习结果显示新增和更新数量。
- [ ] 状态提示自动消失，但错误与待处理项保持可见。

验收：每个成功提示都与页面实际变化一致。

### P1：重构侧边栏

- [x] Tab 改为“我的资料 / 导入资料 / 设置”。
- [x] 顶栏只保留品牌、同步/刷新和当前页面相关动作。
- [x] 当前档案选择器变得更紧凑，并在 popup 与侧栏使用同一名称。
- [x] 资料按大类分组，每类一个容器，不给每条资料套独立卡片。
- [x] 常用值保持清晰，标题、别名和网页信号只在编辑时出现。
- [x] 空状态提供导入资料和手动添加入口。

验收：用户能在 10 秒内找到并修改一条邮箱或签证资料。

### P1：简化导入资料

- [x] 文件导入保持默认入口，手动录入作为第二入口。
- [x] 文件选择后突出文件名、格式、大小和更换入口。
- [x] 分析前说明会生成建议，不会自动覆盖资料。
- [ ] 分析结果进入确认页，允许逐项选择、修改和取消。
- [ ] 明确区分“没有提取到内容”“AI 未确认内容”“文件格式不支持”。

验收：用户始终知道当前文件状态、分析状态和保存状态。

### P1：渐进式 AI 设置

- [x] 未配置时显示三步设置引导。
- [x] 已配置时收起介绍，只显示服务商和连接状态。
- [ ] 增加“测试连接”。
- [x] 模型与自定义指令放入高级设置。
- [x] 语言和外观移入“通用设置”小节。
- [ ] 在触发 AI 的位置说明会发送哪些字段和资料。

验收：用户能判断 AI 是否可用，并理解 Key 与资料的数据流向。

### P2：填写控制与恢复

- [ ] 网页字段依次显示“准备填写、已填写、未匹配”状态。
- [ ] 填写后提供短时“撤销本次填写”。
- [ ] 身份证件、财务和健康等敏感字段可单独确认。
- [ ] 页面 DOM 变化时重新匹配并明确反馈。
- [ ] AI 请求可取消，超时后可重试。

验收：用户可以停止、撤销并理解部分完成的填写过程。

### P2：无障碍与视觉统一

- [x] 所有图标按钮都有可访问名称和 Tooltip。
- [ ] 键盘可完成主要旅程，焦点顺序与视觉顺序一致。
- [ ] 浅色、夜间模式均满足可读对比度。
- [x] 360px 侧栏与 410px popup 不横向溢出。
- [x] 英文长文案不截断、不推动控件。
- [x] 动画不引发布局变化，并遵循减少动态效果设置。

验收：Chrome 键盘操作、中文/英文、浅色/夜间模式全部完成视觉回归。

## 6. 数据与技术配合

UI 升级需要以下结果结构支持：

```js
{
  filled: 7,
  unmatched: 3,
  skipped: 1,
  failed: 0,
  details: [
    { index: 0, status: "filled", source: "local" },
    { index: 1, status: "unmatched", reason: "no suitable saved detail" }
  ]
}
```

资料条目后续应增加稳定 `id`，为站点记忆、逐项结果、撤销和编辑提供可靠引用，避免继续依赖数组下标。

## 7. 验证方式

每个阶段必须同时完成：

1. 自动测试：结果计数、零填充、AI 失败、资料合并。
2. Fixture 测试：文本、选择题、自定义按钮、文件字段和 iframe。
3. Chrome 实测：招聘网站与签证表格各至少两个。
4. 视觉回归：中文/英文、浅色/夜间、360px/430px。
5. 键盘检查：Tab、Enter、Space、Escape 和焦点可见性。

## 8. 推荐执行顺序

1. Popup 主流程与文案。
2. 结果数据结构与可信反馈。
3. 侧边栏信息架构。
4. 文件导入确认流程。
5. AI 设置渐进化。
6. 撤销、敏感字段确认与网页内状态。
7. 无障碍与完整视觉回归。
